## description
分析用户简历与目标 JD 的匹配度。当用户要求"分析匹配度""看看简历和JD匹配吗""计算匹配分数""简历适不适合这个岗位"时触发。流程：读取简历 → 解析 JD → 计算四维度评分 → 输出分析报告和优化建议。

## 适用场景
- 用户想了解简历与某个岗位的匹配程度
- 用户想在投递前评估简历的竞争力
- 用户想获取简历优化方向

## 评分体系（总分 100）

| 维度 | 分值 | 计算方式 |
|------|------|----------|
| 技能匹配 | 40 | 简历与 JD 的关键词交集，每个匹配词 8 分，封顶 40 |
| 经验匹配 | 30 | 实习经历数×10 + 项目经历数×5，封顶 30 |
| 学历匹配 | 15 | 基于学历等级评估 |
| 关键词匹配 | 15 | 简历对 JD 关键词的覆盖率，每个匹配词 3 分，封顶 15 |

## 工作流程

### 步骤① — 获取简历数据

用 `read_db` 读取用户简历：

```
工具：read_db
参数：{ "table_name": "resumes", "query": { "user_id": 当前用户ID } }
```

如果返回为空，提示用户先生成简历。

### 步骤② — 获取 JD 数据

确认 JD 来源：
- 用户已提供 JD 文本 → 调用 `parse_jd_tool` 解析
- 用户提供了链接 → 调用 `parse_jd_tool` 解析
- 用户未提供 → 主动询问

### 步骤③ — 计算匹配度

收集到简历数据和 JD 数据后，调用 `calculate_match_score` 函数计算：

```python
from agent.services.match_score import calculate_match_score

result = calculate_match_score(jd_data, resume_data)
```

**输入**：
- `jd_data`：JD 解析结果（含 requirements、responsibilities 等字段）
- `resume_data`：简历数据（含 internship_exp、project_exp、personal_strengths 等字段）

**输出**：
- `total_score`：总分（0-100）
- `skill_score`：技能匹配分（0-40）
- `experience_score`：经验匹配分（0-30）
- `education_score`：学历匹配分（0-15）
- `keyword_score`：关键词匹配分（0-15）
- `matched_keywords`：已匹配的关键词（前 10 个）
- `missing_keywords`：缺失的关键词（前 10 个）
- `suggestions`：优化建议列表

### 步骤④ — 输出分析报告

按以下结构向用户展示结果：

**总览**：
- 总分及等级判定（80+ 优秀 / 60-80 良好 / 60 以下需优化）
- 一句话总结匹配情况

**分维度解读**：
- 技能匹配：匹配了哪些关键词，缺失哪些
- 经验匹配：实习/项目数量是否充足
- 关键词覆盖：JD 核心要求的覆盖程度

**优化建议**：
- 优先展示影响分数最大的优化方向
- 具体到"在简历中补充 XX 关键词""增加一段 XX 相关的实习经历"
- 不泛泛而谈，每条建议可直接行动

### 步骤⑤ — 联动优化（可选）

如果用户希望根据分析结果优化简历：
- 引导用户使用 resume_gen skill 修改简历
- 针对缺失关键词，建议在对应经历描述中补充
- 修改完成后可再次调用本 skill 验证分数变化

## 注意事项
- 如果 JD 数据中 requirements 或 responsibilities 为空，提示用户提供更完整的 JD
- 匹配度计算依赖关键词提取，对中文岗位（产品、运营等）的匹配精度有限，需向用户说明
- 建议是基于算法的客观分析，但最终简历策略需结合用户实际情况判断
