## description
生成或修改用户的预览版简历。当用户输入"写简历""修改简历""帮我生成简历""更新简历"等与生成或修改简历相关的请求时触发。流程：读取现有简历 → 解析 JD → 引导用户补充经历 → 整理生成 → **保存到数据库**。

## 适用场景
- 用户首次生成简历
- 用户要求针对新 JD 重新生成简历
- 用户要求修改已有简历的某个模块（如补充实习经历、修改项目描述）

## 数据结构

简历存储在 `resumes` 表，按 `user_id` 唯一，包含以下 JSON 字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| basic_info | dict | 姓名、手机、邮箱、学校、专业、学历、GPA、毕业年份 |
| education | list | 教育经历列表 |
| internship_exp | list | 实习经历列表，每项含 company/title/time/description |
| project_exp | list | 项目经历列表，每项含 name/role/time/description |
| personal_strengths | list | 个人优势/技能列表（3-5 条，综合能力导向） |

## 工作流程

### 步骤① — 读取现有数据

先用 `read_db` 工具读取用户当前简历：

```
工具：read_db
参数：{ "table_name": "resumes", "query": { "user_id": 当前用户ID } }
```

- 如果返回数据不为空，说明用户有已有简历，展示当前内容供用户确认修改范围
- 如果返回为空，说明用户首次生成，进入完整流程

### 步骤② — 获取 JD

确认用户目标岗位的 JD：
- 用户已有 JD 文本 → 直接使用
- 用户已提供 JD 文本 → 直接使用
- 用户提供了链接 → 调用 `parse_jd_tool` 解析
- 用户未提供 → 主动询问

### 步骤③ — 引导提问

按以下维度逐步提问（每轮 2-3 个问题）：
- **JD 需求映射**：将 JD 要求转化为具体经历问题。如 JD 要求"数据分析能力"→ 问"有没有数据分析相关项目？背景、职责、工具是什么？"
- **查漏补缺**：JD 看重但用户未提及的领域，主动追问
- **个人优势**：GPA、排名、竞赛获奖、技能证书

### 步骤④ — 整理生成

收到用户内容后，按以下规范整理：

**基本信息 basic_info**：
- 必填：姓名、手机、邮箱、学校、专业、学历
- 985/211/双一流高校须在校名后添加标签
- GPA 仅在排名专业前 20% 时展示

**教育经历 education**：
- 按时间倒序排列
- 包含学校、专业、学历、时间段

**实习经历 internship_exp**：
- 按时间倒序，优先展示与 JD 最相关的
- 每段实习的 description 不超过 5 条
- 采用 STAR 格式 + 四字概括前缀（如"搭建框架：xxx"）
- 描述向 JD 关键词对齐，不改变用户提供的事实

**项目经历 project_exp**：
- 选 2-3 个与 JD 最相关的项目
- 每个项目的 description 不超过 5 条
- 同样采用 STAR 格式 + 四字概括前缀

**个人优势 personal_strengths**：
- 3-5 条，聚焦综合能力（不生成"专业技能"子板块）
- 如数据分析、沟通协作、快速学习等

**总体约束**：
- 内容控制在 A4 单页以内
- 信息不足时可编造合理内容，标记 [建议确认]
- 不改变用户提供的事实

### 步骤⑤ — 保存到数据库

用 `edit_db` 工具保存：

```
工具：edit_db
参数：{
  "table_name": "resumes",
  "query": { "user_id": 当前用户ID },
  "data": {
    "basic_info": {...},
    "education": [...],
    "internship_exp": [...],
    "project_exp": [...],
    "personal_strengths": [...]
  }
}
```

`edit_db` 会自动判断：记录存在则更新，不存在则插入。同时会自动记录历史版本（`resume_history` 表）。

### 步骤⑥ — 确认与微调

保存后展示最终简历内容，询问用户是否需要调整。如需修改，针对具体字段重复步骤④⑤。

## 部分修改流程

当用户只要求修改某个模块（如"帮我改一下实习经历的描述"）时：
1. 读取现有简历（步骤①）
2. 只修改用户指定的字段，其余保持不变
3. 用 `edit_db` 保存整个 data 对象（未变更的字段原样写入）
